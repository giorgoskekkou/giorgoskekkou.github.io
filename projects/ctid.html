Cyber Threat Incident Dashboard (CTID)

Programming Principles Project

Summary & Screenshots

This is a dashboard that can be used by network operators to monitor cybersecurity incidents. It offers an interactive and user-friendly Graphical User Interface (GUI), including helpful pop-ups. The GUI is well-structured and organized, with an intuitive menu for accessing the various modes. It organizes and stores the huge dataset, whose input CSV file contains 200k+ rows of data, in an SQL-based database. By choosing the requested sample rate (in the order of 10), the dashboard user can search for the top 15 incidents based on the malware category and give out reports for the number of incidents. It can also search for specific incidents based on the given filters or the ID, and it can add new incidents to the database. Prior to implementing this project, there was a design of the UML class diagram and use case diagram.

Skills: Java programming, OOP, Abstraction, Inheritance, Encapsulation, UML diagrams (class diagram, use case diagram), GUI (JavaFX), Database (MySQL server/workbench, JDBC interface), File Reader, Exception Handling, Event Handling, Popups, Login Screen, Maven
