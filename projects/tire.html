Tire Service on the spot

Software Engineering Project

Summary & Screenshots

This project was a part of the Software Engineering class, which required developing a full-blown software for a company. The main objective of the assigned project was to help with scheduling and managing appointments for tire services. The software development is divided into 4 main phases: Planning, Requirement Analysis, Design and Implementation, while also including Testing throughout the whole project. Users can request their desired service by filling out a form, and then the service will be assigned to a technician by the system admin to start working on it. There are different dashboards for the various company staff, based on providing the appropriate credentials via a login in screen. This is a full-stack web application with the backend developed using the Flask Python framework and SQLite for the database. The frontend consists of the standard HTML, CSS, JavaScript, and jQuery, and it also includes Bootstrap to enhance the visuals. The project is implemented using Object-Oriented Programming (OOP) principles, ensuring modularity, reusability and maintainability of the code.

Skills: Software Engineering, Software Development Live Cycle (SDLC), Web Development/Web Application, Python Programming, Flask(API/Backend), HTML/CSS/JS(jQuery)/Bootstrap (Frontend), Database (SQLite), jinja, OOP, UML diagrams (class and use case), Reports (Define Requirements/Specifications, Analysis, Design, Testing), Pseudocode, Testing (Black Box and White Box Testing)
